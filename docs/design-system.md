# AidSec Design System & UI Guidelines

This document outlines the core design tokens, component library architecture, and responsive guidelines for the AidSec website. It serves as a developer handoff and reference for maintaining strict adherence to the **Swiss Vault** brand identity.

## 1. Design Tokens

The foundational design tokens are generated via `scripts/design_token_generator.py` and are injected into `css/variables.css`.

### Brand Colors ("Swiss Vault")

- **Primary Navy (`--navy`)**: `#0b1d3a`
- **Deep Navy (`--navy-deep`)**: `#060f1f`
- **Gold Accent (`--gold`)**: `#c8a84c`
- **White (`--white`)**: `#ffffff`
- **Background (`--gray-50`)**: `#f8fafc`
- **Success Green (`--green-safe`)**: `#10b981`
- **Warning Red (`--red-danger`)**: `#ef4444`

### Typography Scale (Modular)

- **Display Font (`--font-display`)**: `Instrument Serif` (Used for headers, H1-H3)
- **Body Font (`--font-body`)**: `Plus Jakarta Sans` (Used for body text, UI elements, buttons)
- **Fluid Scaling**: Typography uses CSS `clamp()` to scale fluidly between mobile and desktop viewports, ensuring perfect readability and an elegant layout on all devices without media-query snapping.

### 8pt Grid & Spacing

- All spacing (margins, paddings, gaps) strictly follows an **8pt (0.5rem) grid system**:
  - `--space-1`: `0.25rem` (4px)
  - `--space-2`: `0.5rem` (8px)
  - `--space-3`: `0.75rem` (12px)
  - `--space-4`: `1rem` (16px)
  - `--space-8`: `2rem` (32px)
  - `--space-12`: `3rem` (48px)
  - `--space-16`: `4rem` (64px)

## 2. Component System Architecture

Components are built to be modular, reusable, and strictly consume the design tokens.

### Buttons (`.btn`)

- **Primary Gold (`.btn--gold`)**: Uses the `--gold` token for background. Features subtle hover state shifting and a refined glow for high conversion.
- **Ghost/Outline (`.btn--outline`)**: Border-only button for secondary actions.
- **Micro-Animations**: All interactive elements include a `.transition-base` (e.g., `all 0.2s cubic-bezier(0.4, 0, 0.2, 1)`) for micro-interactions.
- **Tap Targets**: Mobile compliance guaranteed globally—buttons and links must maintain a minimum `min-height: 44px` and sufficient padding.

### Cards (`.service-card`, `.article-card`)

- **Glassmorphism**: Subtle `--glass-blur` and translucent borders are used for interactive cards (e.g., `.trust__feature`, `.service-card`).
- **Shadows**: Tokenized drop-shadows range from `--shadow-sm` to `--shadow-xl`, emphasizing visual hierarchy.

## 3. Accessibility (a11y) Compliance

AidSec commits to exceptional accessibility standards (WCAG 2.1 AA/AAA):

- **Contrast Ratios**: Navy backgrounds paired with white or gold text ensure passing contrast ratios.
- **ARIA Navigation**: The global `<header>` includes expansive `aria-` labels (`role="banner"`, `aria-label="Hauptnavigation"`, `aria-expanded`).
- **Semantic HTML**: `<main>`, `<article>`, `<section>`, and `<nav>` tags are used unconditionally.
- **Decorative Elements**: All decorative SVGs require `aria-hidden="true"`.
